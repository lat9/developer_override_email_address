Developer Override Email Address (for Zen Cart v1.5.0 through v1.5.4)

Version 1.0.0

Current Support Thread at Zen Cart Forums: http://www.zen-cart.com/showthread.php?213695-myDEBUG-Backtrace-Support-Thread

What it does

This plugin makes use of the built-in Zen Cart constant: DEVELOPER_OVERRIDE_EMAIL_ADDRESS. When that constant is set to a non-blank value, the zen_mail function sends all emails to the address (or addresses) specified by that constant. That allows you, as a Zen Cart developer, to use "live" data during your testing without sending unwanted emails to real customers!

The plugin should never, ever be used on a live site; its utility is on test-site clones of your live site.

Installation

There are no core-file overwrites or database changes associated with this module. Installation is simple:

  1. Unzip the plugin's package file, rename the YOUR_ADMIN folder to match your custom admin folder's name.
  2. Edit (using a text-editor like Notepad++) the file /includes/extra_configures/developer_override_email_address.php, setting the email address(es) to which all test-store emails will be sent. If multiple addresses are required, separate the addresses using a comma (,).
  3. Copy the plugin's files to your store's file-system:
    1. /includes/extra_configures/developer_override_email_address.php
    2. /YOUR_ADMIN/includes/auto_loaders/config.developer_override_email_address.php
    3. /YOUR_ADMIN/includes/extra_configures/developer_override_email_address_admin.php
    4. /YOUR_ADMIN/includes/init_includes/init_developer_override_email_address.php
    5. /YOUR_ADMIN/includes/languages/english/extra_definitions/developer_override_email_address_message.php

Uninstall

Delete the plugin's files from your store's file-system.
